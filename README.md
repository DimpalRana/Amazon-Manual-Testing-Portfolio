# Amazon Web Application - Manual Testing Portfolio

## 📌 Project Overview
This repository contains my very first manual software testing project. As a fresher entry-level QA professional, I performed functional testing on the Amazon login and security verification flows. The goal of this project was to identify how the system handles valid data, missing fields, and invalid input formats.

## 🛠️ What I Tested
* **TC_Login_01**: Security OTP authentication prompt validation for standard mobile entries.
* **TC_Login_02**: System boundary and error behavior when fields are left completely empty.
* **TC_Login_03**: System validation checks when incorrect 5-digit short mobile strings are provided.

## 📁 Repository Contents
* **Test_Cases_ECommerce_Login.xlsx**: A professionally formatted spreadsheet documenting Test IDs, execution steps, expected behaviors, real-time actual outcomes, and PASS/FAIL testing statuses.

## 🚀 Skills Demonstrated
* Writing structured Test Cases from scratch
* Positive and Negative Software Testing
* UI and Functional Verification
